# Codexでキャラクター設定を追加する運用

このプロジェクトでは、WebフォームではなくCodexへの依頼を主な入力口にする。

## ユーザーからCodexへの依頼例

```text
この情報を demo-character の公式設定候補として取り込んで。
確定できる内容は character.json に反映し、未確定や矛盾しそうな内容は inbox に残して。

（ここに原典、メモ、URL、会話ログ、画像生成結果の評価などを貼る）
```

```text
demo-character の好きな食べ物を「いちご大福」に確定して。
公式サイトとAI向けプロンプトにも反映して。
```

```text
この2つの設定が矛盾していないか確認して。
矛盾していたら、どちらを公式にするべきか判断材料をまとめて。
```

## Codex側の処理方針

最重要ルール: 足りない情報は推測で補完しない。

- 未提供の事実は `未定義` として扱う
- 好きな食べ物、苦手なもの、一人称、家族構成、出身地などを勝手に作らない
- 苗字・名前の形式が明示されていないキャラクター名は、無理に分解しない
- 公式化した内容と未定義の内容を分けて報告する

1. 原典やメモを読む
2. 既存の `character.json` と照合する
3. 内容を分類する

- 確定: 公式設定として `character.json` に反映
- 未確定: `content/inbox/{characterId}/` にメモとして保存
- 矛盾候補: 既存設定との差分を明記して確認待ちにする
- 派生アイデア: 公式化せず、候補として残す

4. `npm.cmd run build` で生成結果を確認する
5. 変更点を簡潔に報告する

## 反映時の原則

- 公式設定にないことは勝手に断定しない
- 足りない設定は推測で補完せず、`未定義` として残す
- 出典がある場合は `sources` に残す
- 迷う内容は `draft` として設定する
- 年齢、身長体重、口調など、AI出力に強く影響する項目は優先して明確化する
- 画像・動画生成に関わる外見、服装、色、持ち物、動きは `promptGuidance` にも反映する
