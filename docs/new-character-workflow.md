# 新規キャラクターページ作成ワークフロー

このドキュメントは、新しいキャラクターページを Codex がユーザと対話しながら作るための手順書です。

## 最重要ルール

- 足りない情報を推測で補完しない。
- 未確定の情報は `未定義` として扱う。
- 任意項目に情報がない場合は、空配列または項目自体を省略してよい。
- キャラクター名を苗字・名前に分割しない。ユーザが明示した場合だけ分割情報を書く。
- 見た目、年齢、性格、関係性、世界観、時系列は、ユーザが出した情報だけを公式設定として扱う。
- 迷った情報は `content/inbox/{characterId}/` にメモとして残し、公式設定には入れない。

## 最初に聞くこと

ユーザに一度に全部聞かない。まず以下だけ確認する。

1. キャラクター表示名
2. URL用ID
3. 1〜3文の概要
4. 既に確定しているプロフィール項目
5. イメージカラー

IDは小文字英数字とハイフンを推奨する。例: `zannenin`, `my-character`

ユーザが大文字、アンダースコア、空白などを含むIDを指定した場合は、ビルド検証に通る英小文字・数字・ハイフン形式へ正規化する。元の指定IDは `content/inbox/{characterId}/README.md` に確認事項として残す。

## 対話の進め方

まずユーザから出た情報をそのまま整理する。

```text
確認できた公式設定:
- ...

未定義のままにする項目:
- ...

追加で聞くとよさそうな項目:
- ...
```

追加質問は最大3つまでにする。ユーザが「まず入れられるところから」と言った場合は、質問を増やさず初期ページを作る。

## 初期作成後の次アクション提案

初めてキャラクターを作った直後は、完了報告だけで終わらず、次に決めるとよい項目を提案する。

提案は、現在の `character.json` を見て未定義や空配列が多い領域から選ぶ。公式設定として勝手に補完せず、「次に聞くとよいこと」として扱う。

優先順の目安:

1. イメージカラー: サイト全体の印象がすぐ変わり、キャラクター固有の雰囲気を決めやすい。`theme.primary`, `theme.secondary`, `theme.accent`, `theme.paper` を中心に反映する。
2. 概要・キャッチフレーズ: トップ表示、OGP、AI向けプロンプトの芯になる。
3. 基本プロフィール: 年齢、身長、好きな食べ物、一人称など、短く確定しやすい項目を埋める。
4. 外見・服装・持ち物: 画像生成AIへの影響が大きいため、確定している要素だけを `settings` と `promptGuidance.image` に反映する。
5. 話し方・性格・世界観: 会話AIやText-to-Text出力の品質に直結するため、断定できる内容だけを整理する。
6. リンク、ビジュアル資料、年表、二次創作ガイドライン: 素材や運用方針が出てきた段階で追加する。

特に初期情報が名前とIDだけの場合は、まずイメージカラーから聞くのを推奨する。質問例:

```text
次はイメージカラーから決めるのがおすすめです。ユズの色として、以下を教えてください。

1. メインカラー:
2. サブカラー:
3. アクセントカラー:

色名だけでも、カラーコードでも大丈夫です。迷っている色は候補として inbox に残します。
```

次アクションの提案では、質問を最大3つまでにする。ユーザが「おすすめで進めて」と言った場合でも、未提供の事実は公式化せず、色の候補や方向性は `draft` または inbox の確認待ちとして扱う。

## 作成するファイル

新規キャラクターIDを `{characterId}` とする。

```text
content/
  characters/
    {characterId}/
      character.json
      assets/
        brand/
        links/
  inbox/
    {characterId}/
      README.md
```

画像がまだない場合、`assets/` は空でもよい。

## character.json の初期形

最低限、次の形で始める。

```json
{
  "id": "{characterId}",
  "displayName": "{キャラクター名}",
  "catchphrase": "未定義",
  "summary": "{ユーザが提供した概要。なければ未定義}",
  "theme": {
    "primary": "#151217",
    "secondary": "#d4a72c",
    "accent": "#fff3c4",
    "paper": "#fffaf0",
    "panel": "#ffffff",
    "text": "#1f1a12",
    "muted": "#756b5e"
  },
  "profile": {
    "呼称": "{キャラクター名}",
    "年齢": "未定義",
    "身長": "未定義",
    "体重": "未定義",
    "好きな食べ物": "未定義",
    "苦手なもの": "未定義",
    "一人称": "未定義"
  },
  "likes": [],
  "links": [],
  "contentLinks": [],
  "glossary": [],
  "settings": [],
  "sideFlavors": [],
  "visualReferences": [],
  "timeline": [],
  "promptGuidance": {
    "agent": [
      "公式設定にないことは断定しない。",
      "未定義の項目は必要に応じてユーザに確認する。"
    ],
    "t2t": [
      "キャラクター情報を参照資料として扱い、未定義の設定は補完しない。"
    ],
    "image": [
      "外見設定が未定義の場合は、未定義として扱い、勝手に補完しない。"
    ],
    "video": [
      "動きや演出が未定義の場合は、未定義として扱い、勝手に補完しない。"
    ]
  },
  "sources": []
}
```

イメージカラーが未定義の場合は、上記のデフォルトテーマでよい。ただし「キャラクター固有の色」とは説明しない。

## 情報の入れ方

### profile

年齢、身長、体重、出身、好きな食べ物、一人称、愛称など、短い項目を入れる。

不明な値は `未定義`。

### glossary

用語、組織、地名、サービス名、固有名詞を入れる。

```json
{
  "term": "用語名",
  "definition": "説明"
}
```

### settings

世界観、性格、話し方、衣装、能力、関係性など、本文として読ませたい設定を入れる。

```json
{
  "title": "設定名",
  "body": "設定本文",
  "status": "official"
}
```

未確定の場合は `status` を `draft` にする。

### sideFlavors

本筋ではないが面白いサブ設定、ネタ、周辺文脈を入れる。

### timeline

日付がある出来事を入れる。年齢固定キャラクターの場合は、作中時間と現実の活動履歴を混ぜない。

- `timelineType: "fictional"`: 架空年表。キャラクター内時間、作中前史、自己申告の誕生や成長など。
- `timelineType: "real"`: 実年表。公開日、SNSでの出来事、活動シーズン、サービス終了など。

```json
{
  "date": "2026-04-29",
  "event": "出来事",
  "detail": "補足",
  "timelineType": "real"
}
```

期間は `2026-04-27〜2026-04-28` のように書く。

年齢が固定の場合、`0歳頃`, `12歳頃` のような出来事は `fictional` に入れる。実時間の経過で現在年齢を増やさない。

### links / contentLinks

SNSは `links`、公式ゲーム・Discord・資料集・ショップなどは `contentLinks` に入れる。

```json
{
  "label": "YouTube",
  "url": "https://example.com",
  "type": "social",
  "displayUrl": "YouTube"
}
```

### visualReferences

画像がある場合だけ入れる。

```json
{
  "label": "三面図",
  "path": "assets/outfit-reference.png",
  "type": "outfit",
  "description": "通常衣装の三面図。"
}
```

画像ファイルは `content/characters/{characterId}/assets/` 以下に置く。

## 二次創作ガイドライン

ユーザが希望した場合だけ `fanworkGuidelines` を追加する。

寛容寄りの初期案が必要な場合は、次の方針でスキャフォールドする。

- 二次創作、AI生成、ファンアート、動画、配信、記事化を許可寄りにする。
- 商用利用は応相談。
- メディアサイト、動画投稿サイト、SNSなどのPVや再生数に応じた広告収入は許可。
- 公式と誤認させる表示は禁止。
- キャラクターや第三者を傷つける使い方は禁止。

## 作業手順

1. `content/characters/{characterId}/character.json` を作る。
2. `content/inbox/{characterId}/README.md` を作り、未整理情報や今後確認したいことを書く。
3. 必要なら画像を `assets/` に置く。
4. `npm.cmd run build` を実行する。
5. `npm.cmd run check` を実行する。
6. ローカルで `http://127.0.0.1:4173/{characterId}/` を確認する。
7. 変更内容、未定義のままにした項目、次に聞くとよい項目をユーザに報告する。
8. 初回作成時は、次アクションとしてイメージカラー、概要・キャッチフレーズ、基本プロフィールなどから最大3つを提案する。

## ユーザへの確認テンプレート

```text
まず初期ページを作るために、以下だけ確認します。

1. キャラクター名:
2. URL用ID:
3. 1〜3文の概要:
4. 確定しているプロフィール:
5. イメージカラー:

未確定の項目は推測せず、未定義のまま進めます。
```

```text
3. アクセントカラー:
```
